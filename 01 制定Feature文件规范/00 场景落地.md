## 测试用例转 Feature 文件

我现在有800条测试用例（用例文本不是BDD格式的，粗粒度有粗有细），想要通过AI转换为BDD格式的测试用例，如果让AI在转换过程中，遵循这个规则呢？

```
第一步：抽取 20 条有代表性的（过粗/过细/合适各取几条）
        → 跑一次转换，人工校验输出质量
        → 调整 System Prompt 的 few-shot 示例

第二步：分批处理，每批 30–50 条
        → AI 输出 + 颗粒度标注
        → 人工 review 标注为"拆分"或"抽象"的条目

第三步：将验收通过的 Scenario 按 Feature 边界归组
        → 这一步可以再跑一次 AI（给它看已有 Scenario 列表，让它做归组建议）
```
