UI 自动化批跑
     ↓
捕获接口请求（HAR / Playwright network / mitmproxy）
     ↓
与 UI BDD Feature 文件做对齐
     ↓
推导接口参数依赖关系
     ↓
生成 API BDD Feature 文件（引用 Jinja2 模板）



具体生成流程
```
UI 批跑
  → environment.py 按 Scenario 边界捕获请求
  → scenario_requests.json

                    ↓

推导脚本（ui_to_api_bdd.py）
  输入1: scenario_requests.json
  输入2: UI Feature 文件（提供业务语境和参数值）
  输入3: Jinja2 模板目录（确定哪些参数需显式传入）

  处理：
    接口 URL → 匹配模板
    请求体参数 → 对照模板默认值 → 筛出需显式声明的参数
    UI Scenario 名称 → 推导 API Scenario 名称
    模板 checkpoint → 推导 Then 的业务断言文本

                    ↓

生成 API Feature 文件
  所有 Step 带 【API】前缀
  业务语言，不暴露 HTTP 细节
  参数依赖全部由模板 + context.vars 承接
```
